# Project Brief

## One-line Pitch

A casual fantasy board adventure where players roll dice to explore magical maps, land on storybook tiles, collect rewards, overcome light encounters, and unlock new regions.

## Product Identity

- Title: Fantasy Dice Tales
- Tagline: Roll through a living storybook world.
- Primary fantasy: Becoming a small hero party traveling across magical lands.
- Core promise: Every dice roll moves the player toward discovery, reward, or surprise.
- Emotional hook: Anticipation, charm, collection, map completion.
- Audience fantasy fulfilled: A low-friction fantasy adventure that feels playful, collectible, and progress-rich.

## Business Framing

- Platform: Mobile
- Genre: Casual board-game adventure
- Session target: 3-7 minutes for normal session, 30-60 seconds for micro session
- Revenue model: F2P with optional IAP, rewarded video, event pass or bundles later
- IAP anchor: Dice packs, energy refill, event pass, cosmetic/collection bundles
- Prototype timeline: Validate core dice-board loop first before economy/social expansion

## Design Constraints

Must-have pillars:

- Dice movement as core action
- Board map exploration
- Tile-based events
- Fantasy cartoon-border presentation
- Clear reward/progression feedback

Nice-to-have pillars:

- Branching paths
- Character helpers
- Map collection book
- Weekly event boards
- Light combat or encounter resolution

Forbidden directions for prototype:

- Slot-machine primary loop
- Complex RPG combat
- Heavy social attack/steal economy
- Large liveops economy before core loop validation

Engine / technical constraints:

- Mobile-first UI
- Data-driven board and tile definitions
- Server-authoritative economy if production
- Prototype can be local/offline for validation

Content budget constraints:

- Prototype should require one board, one dice set, 8-12 tile types, 1-2 characters, and limited reward assets.

## Success Criteria

- Retention target: Not final; prototype should test repeat-roll desire.
- Playtest target: 70%+ of testers understand roll -> move -> tile -> reward without explanation.
- Production target: Team can generate new boards from data with minimal engineering.
- Design target: Player always understands current tile, next objective, and reward value within 3 seconds.
