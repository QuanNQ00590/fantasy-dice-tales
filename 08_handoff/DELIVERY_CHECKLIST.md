# Delivery Checklist

## Purpose

Use this checklist before handing the package to dev, art, analytics, external partners, or leadership review.

## Source Package Checklist

- `README.md` reflects the current folder structure.
- `08_handoff/HANDOFF_INDEX.md` reflects the current team entry points.
- Every source-of-truth document has frontmatter with `doc_id`, `version`, `status`, `owner`, and `last_updated`.
- Every critical open decision is captured in `00_meta/DECISION_LOG.md`.
- Prototype scope matches current roadmap and prototype build.

## Team Package Checklist

- Engineering receives: scope, system docs, data schema, tracking plan, and prototype reference.
- Art / UX receives: board system, UI UX, art direction, prototype reference, and asset priorities.
- Analytics receives: tracking plan, schema context, prototype scope, and success metrics.
- Production receives: roadmap, open questions, decision log, and dependency view.

## Handoff Quality Checklist

- Source-of-truth order is clear.
- File names are stable and readable.
- Document owners are named.
- Open questions that still matter are explicitly visible.
- Each team can identify what is approved versus exploratory.
- Prototype success criteria are measurable.
